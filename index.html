Uvod

  • JavaScript je skriptni jezik koji omogućuje primjenu kompleksnih značajki na samoj mrežnoj stranici
  • Plasiran je na tržište 1996. godine
  • Uz HTML i CSS, smatra se temeljem oblikovanja mrežnih stranica
  • Primjena u dinamičnom modificiranju HTML-a i CSS-a putem DOM-a
  • JavaScript se primjenjuje u više od 98,7% mrežnih stranica u sklopu klijentske strane
  • Klijent strana predstavlja sve što se događa na korisnikovoj strani uređaja
  • Potreba za kompleksnijim aplikacijama dolazi do razvoja jednostraničnih aplikacija ili SPA - Single-Page Application
  • Naglasak na "asinkronizirano" ažuriranje - bez osvježavanja cijele stranice
  • Zamjećuju se ograničenja JavaScript-a, te kao posljedica dolazi do razvoja radnih okolina ili "framework"-ova

Radne okoline

  • Radna okolina je skup komponenti programske podrške namijenjeni višestrukom korištenju
  • Sadržavaju module koda za višestruku uporabu, poznate kao biblioteke ili "libraries"
  • Svaka pojedinačna radna okolina može primijeniti zasebnu arhitekturu, kako bi se razvitak težio prema standardiziranim načelima
  • JavaScript radne okoline su razvijene kako bi se pružao unaprijed definiran kod i nadomjestila potreba razvoja vlastitih rješenja
  • Ekosustav i zajednica JavaScript-a omogućuje razvoj radnih okolina koje su primjenjive u različitim područjima (testiranje, razvoj klijent i server strane, razvoj moblinih aplikacija...)
  • Najpopularnije radne okoline u okviru razvoja klijent strane su: , , i Svelte (fa nema logo:/)
  • Zbog daljnjeg razvoja, JavaScript radne okoline se počinju primjenjivati u području razvoja server strane aplikacija
  • Server strana uključuje server, koji dostavlja dinamički sadržaj mrežnim stranicama kao odgovor na korisnikov zahtjev
  • Najpoznatiji alat je runtime environment Node.js , primjenjuje JavaScript za pisanje koda unutar server strane
  • Primjenom "asinkronizacije", Node.js gradi skalabilne mrežne aplikacije
  • Najpopularnije radne okoline (gledajući metrike GitHub zvjezdica) za Node.js i razvoj server strane su: Express i NestJS
  • Arhitektura: MVC, FullStack MVC i REST API

Klijent strane radne okoline

React

  • Razvili su ju Facebook (današnji Meta )
  • JavaScript biblioteka, razvija komponente UI-a namijenjene višestrukom korištenju
  • Omogućuje razvoj kompleksnijih aplikacija, koje primjenjuju asinkroniziran način rada (bez potrebe za osvježivanjem stranice)
  • "Renderira" na strani servera uz pomoć Node.js, te pruža podršku za razvoj mobilnih aplikacija pomoću React Native-a
  • Druge radne okoline direktno komuniciraju s DOM-om te manipulariju stablom DOM-a rezultira usporavanje aplikacije
  • React nudi Virtualni DOM virtualna reprezentacija UI-a pohranjena u navedenom DOM-u (putem biblioteke ofc)
  • Jednostavno shvaćanje pogotovo primjenom JSX-a = sintaksa identična XML-u
  • Component-based radna okolina komponenta predstavlja jedinicu koja gradi UI
  • Hooks su isto ključna značajka React-a, jer pomoću njih se upravlja korisničkim stanjem ili user state
  • Daljnjim razvojem React-a, pojavljuje se i Next izrada full-stack aplikacija

Vue

  • Razvio ju Evan You, kao alternativa React-u
  • Progresivna radna okolina namijenjena izgradnji UI-a (dodatak HTML-u)
  • Središte se temelji na "view" sloju (MVC-a) pogodno za integraciju s postojećim bibliotekama
  • Varijable su reaktivne , što znači da su prilagodljive promjenama u deklarativnom obliku
  • Reaktivni sustav funkcionira na temelju dubokog konvertiranja običnih JS objekata u relativne zastupnike
  • Vue također nudi deklarativno renderiranje gdje se nastavlja standardni HTML sa uzorkom sintakse

Angular

  • Razvili su ju Google
  • Angular isto kao i sve ostale radne okoline, služi kao "view" unutar MVC-a
  • Angular primjenjuje direktive (prefiks 'ng-') povezivanje podataka na view putem kontrolera
  • Glavni temelj Angulara jest dvosmjerno vezivanje podataka automatski način ažuriranje view-a kadgod se model promijeni
  • Dvosmjerno vezivanje je pogodno pri sinkroniziranom čuvanju podataka komponenti

Server strane radne okoline

Express

  • Minimalna i fleksibilna radna okolina Node.js-a razvoj mrežnih aplikacija
  • Primjenjuje različite HTTP protokole kako bi se kreirali brzi i jednostavni API
  • Važni features Express-a su ruter (router), međuprogram (middleware), zahtjev i odgovor (request and reponse)
  • Rutiranje podrazumijeva krajnje točke i kako one odgovoravaju nakon korisnikovog zahtjeva
  • app metoda definira rutiranje HTTP metode (GET, POST, PUT i DELETE)
  • Međuprogram ili middleware je skup funkcija koje imaju pristup na tražene objekte, odgovore i iduću funkciju unutar aplikacijskog ciklusa
  • Odgovor ili response predstavlja HTTP odgovor koji Express šalje zauzvrat nakon dobivenog zahtjeva
  • Zahtjev ili request predstavlja HTTP zahtjev i sadrži značajke samog zahtjeva

Nest

  • Namijenjena izgradnji skalabilnih Node.js aplikacija koje se izvode na strani servera
  • Inkorporira progresivni JavaScript, podržava TypeScript i primjenjuje principe OOP-a, FP-a i FRP-a
  • Dekoratori se smatraju glavnom značajkom Nest-a služe kao označitelji koji definiraju drukčije komponente unutar aplikacije
  • @controller, @catch, @injectible i @module prati IOC ili inverziju kontrole tako što vrši DI ili injektiranje ovisnosti
  • Povezivanje drukčijih slojeva aplikacije i upravljanje ovisnostima putem cijele baze koda
  • Ciklus zahtjev-odgovor ima nekoliko koraka koji su važni za uspješan ciklus
  • 1. Slanje zahtjeva 2. Implementacija sučelja međuprograma 3. Implementacija "čuvara" ili guard-a 4. NestInterceptors ili hvatači za logiranje 5. Protok do kanala 6. Pozivanje rute kontrolera
